# 자기소개서

## 1. 지원동기 및 포부: 서비스의 신뢰를 지탱하는 견고한 뼈대

> "안정적인 서버 구조는 보이지 않지만, 서비스의 신뢰를 지탱하는 가장 중요한 기반입니다."

 차세대 산업혁명의 시대, 혁신적인 서비스들이 매일 쏟아지고 있지만 그 근간이 되는 시스템이 불안정하다면 어떠한 혁신도 모래성에 불과합니다. 저는 지난 4년 9개월간 백엔드 개발자로 근무하며, 보이지 않는 곳에서 데이터의 흐름을 원활하게 하고 서비스의 뼈대를 튼튼하게 구축하는 일에 큰 보람을 느껴왔습니다.

 개발자는 코드를 짜는 사람이기도 하지만, 문제를 가장 먼저 발견하고 해결하는 최후의 보루이기도 합니다. 예기치 않은 서버 오류나 배포 중단 이슈가 발생했을 때, 누구보다 빠르게 상황을 파악하고 긴급 패치와 사후에는 원인을 상세히 분석하고 팀 내 공유 문서로 정리하여 재발 방지 대책을 마련하며 시스템의 전반을 책임지는 엔지니어로 성장해 왔습니다. 입사하게 된다면, 단순히 주어진 기능을 구현하는 것을 넘어 서비스 전체의 흐름을 이해하고 비즈니스 가치를 높이는 개발자로서, 어떠한 상황에서도 흔들림 없이 안정적으로 작동하는 서버 환경을 구축하는 데 기여하겠습니다.

## 2. 직무 경험 및 문제 해결: 제한된 자원의 한계를 돌파한 아키텍처 최적화

이전 회사에서 IoT 전광판 백엔드 시스템 전반을 담당하며, TypeScript와 Python(Flask) 기반의 REST API 서버, Node.js 기반의 MQTT 서버 개발 및 MySQL 데이터베이스 설계, AWS 환경 배포를 주도했습니다. 특히 AWS 환경에 CI/CD 파이프라인을 구축하여 배포 프로세스를 자동화하고 개발 생산성을 크게 향상시켰습니다.
가장 기억에 남는 성과는 단일 서버 환경에서 발생한 심각한 트래픽 병목과 API 응답 지연 문제를 구조적으로 해결한 경험입니다. 당시 특정 시간대에 대량의 크론잡 배치 작업과 복잡한 사용자 API 요청이 동시다발적으로 쏟아지며 프로세스가 고갈되는 현상이 발생했습니다. 저는 단순히 서버 인프라를 추가 증설하여 비용으로 문제를 덮는 대신, 시스템 내부 자원의 효율적 활용을 목표로 삼았습니다.

무거운 작업은 Celery와 Redis를 도입해 백그라운드 워커(Worker)로 비동기 분리하고, 내부 호출 구조를 분석해 불필요한 중복 쿼리를 제거했습니다. 동시에 적극적인 Redis 캐싱을 통해 DB 부하를 줄이고, 이에 맞춰 uWSGI 환경 설정을 세밀하게 튜닝했습니다. 그 결과, 추가적인 비용 지출 없이 타임아웃 오류를 완전히 해소하고 안정적인 API 응답 속도를 확보할 수 있었습니다. 이 경험을 통해 '진정한 문제 해결'이란 무작정 자원을 늘리는 것이 아니라, 문제의 본질을 파악하고 아키텍처를 최적화하는 것임을 깊이 깨달았습니다.

## 3. 성격의 장단점: 엣지 케이스를 놓치지 않는 '기분 좋은 걱정'

 저의 단점은 다소 사소한 걱정이 많다는 점입니다. 여행을 갈 때도 비가 오거나 목적지가 문을 닫을 경우 등 발생할 수 있는 여러 돌발 상황을 고려해 대안을 세워두어야 마음이 놓입니다. 실제로 여행 중 비행기 결항이라는 초유의 사태가 발생했을 때도, 미리 준비해 둔 비상금과 여벌 옷 덕분에 위기를 유연하게 넘길 수 있었습니다.

 하지만 이러한 성향은 백엔드 개발 업무에 있어서 시스템의 안정성을 책임지는 가장 강력한 무기가 됩니다. 개발 단계부터 '동시에 여러 요청이 들어오면 어떻게 될까?', '이 데이터가 누락되면 시스템 전체에 어떤 영향을 미칠까?'와 같은 수많은 엣지 케이스를 고민하게 만들기 때문입니다. 앞으로도 이러한 꼼꼼함을 바탕으로, 에러 발생 확률을 최소화하고 효율적이며 방어적인 코드(Defensive Programming)를 작성하여 완벽한 시스템을 구현해 내겠습니다.

## 4. 위기를 도약의 발판으로: 끈기로 이겨낸 시간과 멈추지 않는 성장

 어린 시절 태권도 시합을 준비하며 체급을 맞추기 위해 한여름 히터 앞에서도 땀을 흘리며 한계에 도전했던 기억이 있습니다. 주위의 만류에도 끝까지 포기하지 않고 금메달을 목에 걸었던 당시의 '끈기'는, 제 삶의 중요한 태도로 자리 잡았습니다.

 2023년 3월 이후, 예기치 못한 질병으로 인해 치료와 회복에 전념하고자 잠시 실무를 내려놓아야 했습니다. 하지만 이 기간에도 개발에 대한 열정은 멈추지 않았습니다. 체력이 허락하는 대로 개인 프로젝트를 진행하며 Node.js 기반의 API 서버를 리팩토링하고, Python Flask를 이용한 신규 구조 설계와 비동기 처리 방식을 깊이 있게 학습했습니다. 또한 기술 트렌드 변화에 뒤처지지 않기 위해 Docker, Django, AI Agent 등의 기술을 꾸준히 익히며 개발 감각을 예리하게 유지했습니다.

 몸이 아플 때조차 코드를 읽고 작은 기능을 구현하며 제가 여전히 개발자임을 느꼈고, 이 시간은 단순한 휴식기가 아니라 기술적 성장과 자기 성찰의 귀중한 기회가 되었습니다.

## 5. 주도적인 기술 도전: 한계를 돌파하는 아키텍처 설계 경험

 공백기 동안의 성장은 단순한 이론적 학습에 머물지 않았습니다. 현업 복귀 시 즉각적으로 시스템에 기여할 수 있도록, 대규모 트래픽과 마이크로서비스 아키텍처(MSA)를 고려한 심도 있는 개인 프로젝트들을 직접 기획하고 구현했습니다.

 먼저, 대규모 동시 접속 상황에서 서버 과부하를 방지하기 위해 Redis(ZSET/HASH)와 토큰 버킷(Token Bucket) 알고리즘을 활용한 가상 대기열(Virtual Queue) 시스템을 설계하고 Locust를 통한 부하 테스트로 검증을 완료했습니다. 또한, MSA 환경의 보안과 운영 효율을 높이고자 Nginx API Gateway와 연동된 통합 인증(SSO) 서버를 구축하여 JWT 토큰의 폐기 및 회전(Rotation) 전략을 구현해 냈습니다. 이에 더해 AI 기반 유튜브 레시피 추출 파이프라인을 구축하며 Celery를 활용한 비동기 처리 구조를 완성했고, 이를 Docker Compose와 AWS EC2 기반의 실무형 DevOps 환경에 배포하며 운영 안정성을 체득했습니다. 더 나아가 Django DRF와 Node.js 환경 모두에서 일관된 계층 분리 원칙을 적용한 API 서버 템플릿을 구축하며 아키텍처의 표준화 역량을 길렀습니다.

 이러한 치열한 고민의 결과물들은 저에게 특정 언어나 프레임워크에 종속되지 않고 '왜 이 기술을 써야 하는가?'를 판단하는 설계 능력을 길러주었습니다. 스스로 문제를 정의하고 구조적으로 해결해 낸 이 경험을 바탕으로 어떠한 복잡한 요구사항도 유연하고 안정적인 아키텍처로 구현해 내겠습니다.
