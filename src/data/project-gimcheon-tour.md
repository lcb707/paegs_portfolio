# 김천 관광 안내 웹서비스

빅데이터 활용 디지털옥외광고 시범사업의 일환으로, 키오스크/모바일 기반 관광 정보 서비스를 구축한 프로젝트입니다.
관광지를 홍보하기 위하여 김천 여행지 관련 정보 및 빅데이터 정보를 제공하기 위한 서비스 입니다.

## 프로젝트 개요
- 회사 : (주)이성
- 목적: 김천 관광지 정보와 빅데이터 정보를 사용자에게 직관적으로 제공
- 기간: 2021.04 - 2022.10
- 역할: 백엔드 아키텍처 설계 및 운영

## 주요 역활
- 백엔드 시스템 아키텍처 설계
    - Node.js 기반 백엔드 API 서버 설계/개발
    - AWS DocumentDB(MongoDB) 기반 데이터 저장소 구성
    - Redis 캐시 시스템 구축

- 클라우드 인프라 구축 및 운영
    - AWS EC2 서버 관리 및 운영
    - AWS S3를 활용한 파일 처리 시스템 구축
    - Docker 컨테이너 기반 배포 환경 구성

- DevOps 및 배포 자동화
    - AWS Pipeline을 활용한 CI/CD 파이프라인 구축
    - Shell script 및 Makefile을 이용한 배포 자동화 구현
    - Github 기반 코드 관리 및 버전 관리

## 성과
### 1. [초기 아키텍처 설계] 읽기 중심 트래픽에 대비한 백엔드 구축

- 구축 성과: 관광 정보 및 빅데이터 서비스 특성상 향후 대량의 조회 트래픽이 발생할 것을 예측하여, 기획 단계부터 AWS DocumentDB 앞단에 Redis 캐시 레이어를 배치하는 아키텍처를 설계했습니다. 이를 통해 서비스 오픈 초기부터 피크 시간대에도 지연 없는 안정적인 API 응답 환경을 선제적으로 구축했습니다.

### 2. [DevOps] 배포 파이프라인 자동화 구축

- 구축 성과: 프로젝트 초기 세팅부터 수동 배포로 인한 리소스 낭비와 에러를 방지하고자, AWS Pipeline과 Shell Script, Makefile을 연동한 CI/CD 파이프라인을 구축했습니다. 이를 통해 신규 개발팀이 비즈니스 로직 구현에만 집중할 수 있는 빠르고 안정적인 릴리즈 환경을 조성했습니다.

### 3. [인프라 표준화] Docker 도입을 통한 안정적인 런타임 환경 조성

- 구축 성과: 프로젝트 시작과 동시에 Docker 컨테이너 기반의 배포 환경을 표준으로 채택했습니다. 개발, 테스트, 운영 서버 간의 환경 불일치를 사전에 차단하고, 언제든 동일한 환경에서 배포 및 롤백이 가능한 탄탄한 인프라 기반을 마련했습니다.

### 4. [시스템 확장성] 대용량 미디어 스토리지 독립 구성

- 구축 성과: 다량의 관광지 이미지와 미디어 콘텐츠가 누적될 것을 고려하여, 애플리케이션 서버와 파일 처리 스토리지를 분리 설계했습니다. AWS S3 기반으로 업로드 로직을 구축해 향후 데이터가 기하급수적으로 늘어나더라도 웹 서버에 부하를 주지 않도록 시스템 확장성을 확보했습니다.

### 5. [협업 인프라] 신규 프로젝트를 위한 버전 관리 및 리뷰 체계 확립

- 구축 성과: 백지상태에서 시작하는 프로젝트인 만큼, 초기부터 GitHub 기반의 브랜치 전략과 코드 리뷰 프로세스를 명확히 정립했습니다. 다수의 팀원이 동시에 기능을 개발해도 충돌 없이 이슈를 추적할 수 있는 튼튼한 협업 기반을 다졌습니다.

## 기술 스택
- Node.js, JavaScript
- DocumentDB(MongoDB), Redis
- AWS EC2/S3/Pipeline, Docker
- Shell Script, Makefile
